# 📐 وثيقة التصميم المعماري: نظام استخراج بيانات تليجرام (Telegram Intelligence)

هدفنا هو إضافة قدرة استخراج البيانات من مجموعات تليجرام بنفس كفاءة (أو أفضل من) نظام واتساب الحالي.

## 1. تحليل النهج التقني (Technical Approach Analysis)

أمامنا طريقان لتنفيذ هذه الخاصية، ويجب الاختيار بحكمة:

### الخيار A: المحاكاة عبر المتصفح (Browser Automation) - *مثل واتساب حالياً*
*   **الطريقة:** فتح `web.telegram.org` واستخدام Selenium لسحب البيانات.
*   **العيوب القاتلة:** 
    *   تليجرام ويب يستخدم تقنية **Virtual DOM** معقدة جداً، مما يجعل استخراج أكثر من 50-100 عضو عملية بطيئة وغير مستقرة.
    *   أسماء الكلاسات مشفرة وتتغير باستمرار.
*   **الحكم:** ❌ غير منصوح به للتطبيقات الاحترافية.

### الخيار B: استخدام واجهة برمجة التطبيقات (Native API - MTProto) - *✅ الخيار المقترح*
*   **الطريقة:** الاتصال المباشر بخوادم تليجرام باستخدام بروتوكول MTProto (مكتبة `Telethon` أو `Pyrogram`).
*   **المميزات:**
    *   **سرعة خيالية:** سحب 10,000 عضو يتم في ثوانٍ معدودة (بدون تصفح).
    *   **بيانات عميقة:** القدرة على سحب (User ID, Username, Bio, Last Seen Status) بجانب الاسم.
    *   **الاستقرار:** لا يتأثر بتحديثات شكل الموقع.
*   **المتطلبات من المستخدم:** يحتاج مرة واحدة فقط لاستخراج `API_ID` و `API_HASH` (مجاناً من موقع تليجرام الرسمي) وتسجيل الدخول برقم الهاتف.

## 2. الهيكل الهندسي المقترح (Proposed Architecture)

سيتم بناء وحدة جديدة مستقلة تماماً لضمان عدم التأثير على استقرار نظام واتساب.

### أ. البنية التحتية (Backend Structure)

```plaintext
d:/WORK/2/
├── backend/
│   ├── telegram/              <-- (NEW MODULE)
│   │   ├── __init__.py
│   │   ├── client_manager.py  # إدارة الاتصال والجلسات (Session Management)
│   │   ├── grabber.py         # منطق سحب الأعضاء وفلترة البيانات
│   │   └── config.py          # إعدادات الاتصال
```

### ب. تدفق البيانات (Data Flow)

1.  **المصادقة (Authentication):**
    *   الواجهة تطلب الرقم -> Backend يرسل طلب للتليجرام -> يصل الكود للمستخدم -> يدخله في الواجهة -> يتم إنشاء ملف جلسة آمن `session.session`.
2.  **جلب المجموعات (Fetching Groups):**
    *   يقوم النظام بجلب قائمة المجموعات والقنوات المشترك بها الحساب تلقائياً وعرضها في قائمة منسدلة.
3.  **الاستخراج (Extraction):**
    *   المستخدم يختار المجموعة -> النظام يتصل بالـ API -> يسحب قائمة الـ Participants -> يفلتر من لديهم أرقام هواتف ظاهرة أو Usernames.
4.  **التخزين:**
    *   حفظ البيانات في Excel بتنسيق موحد مع نظام واتساب (`Name`, `Phone`, `Username`, `Bio`).

## 3. المتطلبات التقنية (Requirements)

1.  **مكتبة `Telethon`:** هي المعيار الذهبي للتعامل مع تليجرام في بايثون (Async & Fast).
2.  **تعديل قاعدة البيانات:** إضافة جدول بسيط أو ملف JSON لتخزين الـ Credentials بشكل مشفر محلياً.
