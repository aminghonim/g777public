## 4. Linting & Formatting

While no strict linter is enforced, follow these PEP 8-based guidelines to maintain consistency.

- **Line Length:** Maximum 100 characters.
- **Imports:** Group imports in the following order:
  1. Standard library (`os`, `sys`, `json`)
  2. Third-party libraries (`fastapi`, `nicegui`, `psycopg2`)
  3. Local application imports (`from backend.database_manager import ...`)
- **Naming Conventions:**
  - `snake_case` for variables, functions, and modules.
  - `PascalCase` for classes.
  - `UPPER_SNAKE_CASE` for constants.
- **Docstrings:** Use triple-quote docstrings for all public modules, classes, and functions. Explain what the component does, its arguments, and what it returns.

## 5. Code Style & Architecture

### Config-First Principle
**CRITICAL:** Never hardcode secrets, API keys, URLs, or file paths.
- All configuration must be loaded from environment variables (`.env` file) or the `config/settings.yaml` file.
- Use `os.getenv('MY_VARIABLE', 'default_value')` to access environment variables.
- The `DatabaseManager` and `main.py` are good examples of this principle.

### Error Handling
The established pattern is to use `try...except` blocks to prevent crashes and provide meaningful logs.

- **Catch Specific Exceptions:** Whenever possible, catch specific exceptions (`FileNotFoundError`, `psycopg2.Error`) instead of a generic `Exception`.
- **Logging:** Use the `logging` module for output instead of `print()`. Log errors with context.

### Type Hinting
Use type hints for all function signatures (arguments and return values). This is mandatory for maintaining a clean and predictable codebase.
```python
from typing import Optional, Dict

def get_customer(phone: str) -> Optional[Dict[str, any]]:
    # ... function logic ...
    return customer_data
```

### Modularity
- **UI:** Keep all NiceGUI components within the `ui/` directory. Each major page or component should be in its own file.
- **Backend:** All business logic, database interactions, and external API clients must reside in the `backend/` directory.
- **Database Logic:** The `database_manager.py` is the single source of truth for database connections. All database operations must go through this module. Do not create new database connections elsewhere.
