# 🚀 دليل أوامر Git المختصر (G777 Cheatsheet)

هذا الدليل يشرح لك الأوامر التي ستحتاجها يومياً لحفظ مشروعك ورفعه على GitHub والسيرفر.

---

### 1️⃣ الحفظ اليومي (على جهازك الكمبيوتر)
عندما تنهي تعديلاً ناجحاً وتريد حفظه في "آلة الزمن":

- **الخطوة الأولى: رؤية ماذا تغير**
  ```bash
  git status
  ```
  *(يُظهر لك الملفات التي قمت بتعديلها باللون الأحمر)*

- **الخطوة الثانية: تجهيز الملفات للحفظ**
  ```bash
  git add .
  ```
  *(يجهز كل الملفات المعدلة، ستتحول للون الأخضر في الـ status)*

- **الخطوة الثالثة: تسجيل النسخة (اللقطة)**
  ```bash
  git commit -m "اكتب هنا ماذا فعلت باختصار"
  ```
  *(تم حفظ النسخة الآن داخل جهازك فقط)*

---

### 2️⃣ الرفع للسحاب (GitHub)
لكي تؤمن النسخة أونلاين وتجعلها جاهزة للسيرفر:

- **أمر الرفع:**
  ```bash
  git push origin main
  ```
  *(الآن كودك موجود على حسابك في GitHub)*

---

### 3️⃣ تحديث السيرفر (Linux Server)
لكي يعمل السيرفر بآخر كود رفعته على GitHub:

- **أمر السحب (يُنفذ داخل السيرفر عبر SSH):**
  ```bash
  cd /home/user/g777
  git pull origin main
  ```
  *(الآن السيرفر حصل على آخر التحديثات)*

---

### 4️⃣ أوامر "الإنقاذ" (Emergency)
في حال حدث خطأ وتريد التراجع:

- **التراجع عن تعديلات لم تُحفظ بعد:**
  ```bash
  git restore .
  ```
  *(يمسح كل التعديلات التي قمت بها منذ آخر حفظ ويعيد الكود كما كان)*

- **رؤية سجل النسخ السابقة:**
  ```bash
  git log --oneline
  ```
  *(يُظهر لك قائمة باللقطات السابقة التي سجلتها)*

---

### 💡 نصائح ذهبية:
1. **القاعدة:** (Add -> Commit -> Push) هي الروتين اليومي.
2. **الأمان:** ملف `.env` الذي يحتوي على باسورداتك لن يُرفع أبداً بسب إعدادات الـ `.gitignore`.
3. **التكرار:** احفظ نسخة (Commit) كل ساعة أو ساعتين عمل لضمان عدم ضياع مجهودك.
